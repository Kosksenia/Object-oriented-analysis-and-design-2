# Лабораторная работа №1

## Паттерн «Фабричный метод» (Factory Method): Планировщик логистических маршрутов

### Предметная область

Паттерн **«Фабричный метод»** широко применяется в разработке приложений, где требуется создавать объекты различных типов, но конкретные классы объектов могут меняться или расширяться. Один из примеров — система планирования маршрутов для транспортной компании.

**Планировщик маршрутов** — это компонент приложения, который отвечает за построение оптимальных маршрутов для разных типов транспортных средств с учетом их специфических ограничений (скорость, стоимость, допустимые дороги и т.д.).

### Описание проблемы

В системе логистики существуют различные типы транспортных средств: грузовики, корабли, самолеты. Каждый тип требует своей логики расчета маршрута:

- **Грузовик** — движется по автомобильным дорогам, учитывает ограничения по высоте и весу;
- **Корабль** — движется по водным путям, учитывает осадку, порты, погодные условия;
- **Самолет** — движется по воздушным коридорам, учитывает эшелоны, аэропорты, метеоусловия.

Если клиентский код (например, форма приложения) будет напрямую создавать объекты маршрутизаторов, возникнут проблемы:

1. **Жесткая связанность** — код формы зависит от всех конкретных классов маршрутизаторов;
2. **Сложность расширения** — при добавлении нового типа транспорта (например, электросамоката) придется изменять существующий код;
3. **Дублирование логики** — создание объектов разбросано по всему приложению;
4. **Нарушение принципа Open/Closed** — классы не закрыты для модификации.

### Решение проблемы

Использование **паттерна «Фабричный метод»** для создания маршрутизаторов.

Паттерн гарантирует, что:

1. Создание объектов инкапсулировано в отдельных классах-фабриках;
2. Клиентский код работает только с абстрактной фабрикой и интерфейсами продуктов;
3. Добавление нового типа транспорта не требует изменения существующего кода — достаточно создать новый класс маршрутизатора и новую фабрику;
4. Все маршрутизаторы следуют единому интерфейсу, что обеспечивает полиморфную работу.

## Ключевые особенности реализации

### Динамическое добавление транспорта

В версии с паттерном пользователь может добавлять **любые** типы транспорта через диалоговое окно:

1. Нажать кнопку **"+ Добавить транспорт"**
2. Ввести:
   - Название транспорта (например, "Вертолет", "Велосипед", "Поезд")
   - Среднюю скорость (км/ч)
   - Стоимость за км (руб)
   - Максимальную дистанцию (км)
3. Новый тип мгновенно появляется в выпадающем списке
4. Можно сразу рассчитывать маршруты

### Преимущества реализации

1. **Расширяемость** — новые типы транспорта добавляются без изменения кода
2. **Инкапсуляция** — логика создания скрыта в фабриках
3. **Полиморфизм** — все транспорты работают через единый интерфейс
4. **Гибкость** — пользователь сам определяет параметры транспорта

## Вывод

Паттерн «Фабричный метод» позволил создать гибкую, расширяемую систему, в которой пользователь сам определяет доступные типы транспорта. Это наглядно демонстрирует преимущества паттерна:

- **Слабая связанность** компонентов
- **Возможность динамического расширения**
- **Инкапсуляция логики создания объектов**
- **Возможность добавления новых типов без перекомпиляции**

Версия без паттерна, напротив, показывает все проблемы жесткой связанности и невозможности расширения без изменения существующего кода.
